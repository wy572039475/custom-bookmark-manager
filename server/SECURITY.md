# 安全配置说明

## JWT 密钥配置

### 当前配置
- **JWT_SECRET**: 已更新为 64 位强随机密钥
- **JWT_EXPIRES_IN**: 7 天

### 生成新密钥
如果需要重新生成 JWT 密钥，运行以下命令：

```bash
cd server
node update-env.js
```

然后复制生成的 `JWT_SECRET` 到 `.env` 文件中。

### 密钥要求
- 长度至少 32 字节（64 个十六进制字符）
- 使用强随机数生成器
- 生产环境必须使用随机生成的密钥
- 不要在代码中硬编码密钥

## 数据库密码配置

### 当前配置
- **DB_PASSWORD**: wy318227

### 安全建议
1. 生产环境应使用强随机密码（至少 16 位）
2. 不要使用常见密码或简单模式
3. 定期更换数据库密码
4. 限制数据库用户权限

### 生成强密码
可以使用以下命令生成强密码：

```bash
# 方法1: 使用 OpenSSL
openssl rand -base64 16

# 方法2: 使用 Node.js
node -e "console.log(require('crypto').randomBytes(12).toString('base64'))"
```

## 环境变量管理

### .env 文件
- 包含敏感信息，已加入 .gitignore
- 永远不要提交到版本控制系统
- 生产环境应从安全的地方加载（如环境变量或密钥管理服务）

### .env.example 文件
- 提供配置模板，不包含真实密钥
- 可以安全地提交到版本控制系统
- 新项目部署时参考此文件创建 .env

## 安全最佳实践

1. **定期更换密钥**: 建议每 3-6 个月更换一次 JWT 密钥
2. **使用 HTTPS**: 生产环境必须使用 HTTPS
3. **限制 CORS**: 配置 CORS 白名单
4. **输入验证**: 所有用户输入都应进行验证
5. **SQL 注入防护**: 使用参数化查询
6. **XSS 防护**: 对用户输出进行转义
7. **CSRF 防护**: 实施 CSRF Token 机制
8. **日志记录**: 记录安全相关事件
9. **定期备份**: 定期备份数据库
10. **监控告警**: 设置异常监控和告警

## 密钥轮换

当需要更换 JWT 密钥时：

1. 生成新的 JWT 密钥
2. 更新 .env 文件
3. 重启服务器
4. 所有现有 token 将失效，用户需要重新登录

由于本项目使用自动登录，用户会在下次访问时自动获得新的 token。
