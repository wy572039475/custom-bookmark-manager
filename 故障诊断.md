# 项目诊断报告

## 项目概述
- **项目名称**: 全栈收藏夹系统
- **前端技术**: React 18 + Vite + TypeScript + Tailwind CSS
- **后端技术**: Node.js + Express + TypeScript + PostgreSQL
- **当前状态**: 使用模拟数据,无需数据库即可运行

---

## 可能的问题及解决方案

### 问题 1: 依赖未安装 ❌
**症状**: 启动时提示模块找不到

**解决方案**:
```bash
# 方式1: 使用项目提供的安装脚本
npm run install:all

# 方式2: 分别安装
npm install
cd client && npm install
cd ../server && npm install
```

---

### 问题 2: 后端服务未启动 ❌
**症状**: 前端显示"后端未连接,使用模拟数据"

**解决方案**:
```bash
# 启动后端
cd server
npm run dev
```

后端应该在 `http://localhost:3000` 启动

---

### 问题 3: 前端服务未启动 ❌
**症状**: 无法访问 http://localhost:5173

**解决方案**:
```bash
# 启动前端
cd client
npm run dev
```

前端应该在 `http://localhost:5173` 启动

---

### 问题 4: 路由配置问题 ⚠️
**症状**: 页面空白或404

**解决方案**:
当前路由配置 (App.tsx):
- `/login` → 登录页
- `/register` → 注册页  
- `/` → Dashboard (主页)

如果访问其他路径会显示空白,这是正常的。请访问以上路径之一。

---

### 问题 5: 认证被绕过 ℹ️
**当前状态**: 
- `ProtectedRoute` 组件已被修改为绕过认证
- 可以直接访问 Dashboard 而无需登录
- 这是为了演示和开发方便

如果需要启用认证,恢复 `App.tsx` 中的 `ProtectedRoute`:
```tsx
function ProtectedRoute({ children }: { children: React.ReactNode }) {
  const { user } = useAuth();
  return user ? <>{children}</> : <Navigate to="/login" />;
}
```

## 常见错误及解决

### 错误: "Cannot find module 'xxx'"
**原因**: 依赖未安装
**解决**: 运行 `npm install`

### 错误: "EADDRINUSE: address already in use"
**原因**: 端口被占用
**解决**: 
- 关闭占用端口的进程
- 或修改端口配置 (vite.config.ts 或 server/index.ts)

### 错误: 页面显示白屏
**原因**: 可能是路由错误或代码错误
**解决**: 
1. 打开浏览器开发者工具 (F12)
2. 查看 Console 标签的错误信息
3. 检查 Network 标签,看请求是否成功

## 数据库配置 (可选)

如果需要启用真实数据库功能:

1. 安装 PostgreSQL
2. 创建数据库:
   ```sql
   CREATE DATABASE bookmark_db;
   ```

3. 配置环境变量:
   ```bash
   cd server
   cp .env.example .env
   # 编辑 .env 文件,填入数据库信息
   ```

4. 运行迁移:
   ```bash
   npm run db:migrate
   ```

5. 修改代码,移除模拟数据,使用真实数据库查询
